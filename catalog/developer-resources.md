# 开发工具与工程资源

## 1. Awesome

- 原项目：[sindresorhus/awesome](https://github.com/sindresorhus/awesome)
- 用途：覆盖编程语言、平台、数据库、前端、后端和工具链的Awesome清单总入口。
- 推荐理由：适合作为寻找细分技术资源的第一站。
- 注意：清单收录不代表每个项目都经过安全审计。

## 2. Awesome Python

- 原项目：[vinta/awesome-python](https://github.com/vinta/awesome-python)
- 用途：Python框架、库、工具和学习资源分类清单。
- 适合：Python桌面程序、Web、数据、自动化与AI项目选型。
- 注意：引入依赖前检查维护状态、漏洞记录和许可证。

## 3. Public APIs

- 原项目：[public-apis/public-apis](https://github.com/public-apis/public-apis)
- 用途：公开API分类目录。
- 适合：寻找测试数据、第三方能力和原型接口。
- 注意：免费额度、可用性、隐私政策和商业使用条件可能变化。

## 4. The Book of Secret Knowledge

- 原项目：[trimstray/the-book-of-secret-knowledge](https://github.com/trimstray/the-book-of-secret-knowledge)
- 用途：命令行、网络、系统管理、安全和工程实践资料集合。
- 适合：开发者与运维人员查阅。
- 安全提示：不要在不了解影响时执行系统或网络命令。

## 5. GitHub Gitignore Templates

- 原项目：[github/gitignore](https://github.com/github/gitignore)
- 用途：GitHub维护的多语言与工具 .gitignore 模板。
- 适合：创建新项目时排除构建产物、缓存和本地配置。
- 重要：.gitignore不能替代密钥管理，已经提交的密钥必须立即撤销。

---

选用第三方库前建议检查：最近提交时间、开放Issues、安全公告、依赖树、LICENSE和是否仍在维护。
